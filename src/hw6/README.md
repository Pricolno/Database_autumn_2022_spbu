# Задание №5 (Операторы SELECT и JOIN)

<p>В этой домашке речь пойдет про приют для бездомных животных Тетушки Мэйв. Вот уже несколько лет она помогает зверушкам, и не так давно их количество стало таким большим, что Тетушка уже не помнит обо всех звоих зверях. Именно для этого она обратилась к SQL-специалисту Ивану для создания базы данных, которая поможет ей в ее добром деле. Он их создал, оставил тетушке, и довольный уехал в Стамбул. Ваша задача - помочь Тетушке с ее делами, пользуясь базой данных Ивана.</p>

<p>База данных, кстати говоря, прикреплена ниже.</p>

<h1>Задание 1 (1 балл)</h1>

<p>Так как у Тетушки Мэйв свой большой дом, то логично, что у нее есть коровы. А чтобы они не стояли без дела, Тетушка периодически доит их. А чтобы и ей было удобно, она поделила коров на группы (как - неважно). А теперь она хочет узнать, сколько в среднем она может <q>собирать</q> молока с этих групп.</p>

<p>Задание: Вам необходимо узнать среднее количество молока, которое можно получить с групп, в которых больше 3 коров.</p>

<p>Формат решения: Нужно написать скрипт, который по таблице</p>

<pre><code>create table Cow(
 name bigserial PRIMARY KEY,
 milk_per_second int, -- количество молока в миллилитрах
 breed int -- номер группы
);
</code></pre>

<p>выведет все группы, в которых больше трех коров, а также среднее возможное количество молока в этих группах.</p>

<h1>Задание 2 (2 балла)</h1>

<p>Дома у Тетушки Мэйв живет много животных, включая 100 собак. Поскольку со всеми она гулять не может, она решила сделать так: каждый день она отпускает собак на улицу каждое утро, а вечером ждет их дома. Но поскольку не каждая собака возвращается вовремя, ей хочется понимать, какой именно песик вернется попозже.</p>

<p>Задание: Есть таблица, заполненная номерами от 1 до 100, где одно из чисел пропущено. Необходимо как можно быстрее узнать, какого же номер не хватает, и вывести его.</p>

<p>Формат решения: Нужно представить скрипт, который выведет единственное число, которое отсутствует в таблице <code>Dog</code>.</p>

<pre><code>create table Dog(
 name bigserial primary key
);
</code></pre>

<h1>Задание 3 (3 балла)</h1>

<p>Главная задача Тетушки (но не единственная) - постараться найти хозяев для своих животных. Поскольку с рогатым скотом ситуация особая, она хочет в первую очередь разобраться с кошками. Один аналитик из компании <q>По Коту на Семью</q> сказал ей следующее: &quot;Чтобы узнать спрос на кошку или кота, нужно применить формулу </p>

<p>\[ 
 \frac{sex * \frac{breed^3*count}{age} + (1-sex) * \frac{breed^2*count}{age}}{countOfCats}
\]</p>

<p>где:</p>

<ul>
<li><p>sex - пол (кошка=1, кот=0)</p></li>
<li><p>breed - порода (<code>American=0.5</code>, <code>British=1</code>, <code>Munchkin=1.5</code>, <code>Siberian=2</code>, <code>Bengal=3</code>)</p></li>
<li><p>count - количество цветов кошки</p></li>
<li><p>age - возраст кошки в <strong>месяцах</strong></p></li>
<li><p>countOfCats - количество кошек и котов такой породы у Тетушки Мэйв</p></li>
</ul>

<p>Все очень просто&quot;.</p>

<p>Задание: Ваша задача, как успешного SQL-специалиста, реализовать эту метрику для каждой кошки и отсортировать их по убыванию этой метрики.</p>

<p>Формат решения: Необходимо предоставить скрипт, который по таблице</p>

<pre><code>create table Cat(
 name text, -- имя
 age int, -- возраст в годах
 breed text, -- порода кошки
 sex int, -- пол кошки (F=1, M=0)
 count int -- количество цветов в окраске
);
</code></pre>

<p>Выведет нужную таблицу со столбцами <code>name:text, catGrade:float</code>, отсортированную по убыванию <code>catGrade</code>. Обратите внимание, что <code>breed</code> задан в формате <code>text</code>.</p>

<h1>Задание 4 (4 балла)</h1>

<p>Вернемся к рогатому скоту. У Тетушки есть целых 20 лошадей и коней (суммарно). Будем называть их всех просто лошади. У Тетушки есть целых 4 вида лошадей: <code>Ездовые</code>, <code>Тяжеловесы</code>, <code>Скаковые</code> и <code>Пони</code>. Каждому из этих видов она решила дать свой код длины 3. А тот же Аналитик посоветовал ей дать код и каждой лошади в следущем виде: <code>xxx-aa</code>, где <code>xxx</code> - это код ее вида, и <code>aa</code> - какойто суффикс, характеризующий ее номер. </p>

<p>Тетушка с детства мечатала разводить лошадей, и когда еще жто делать, если не сейчас? Поэтому она решила начать скрещивать лошадей и хочет узнать все возможные комбинации.</p>

<p>Однако у нее есть требования:</p>

<ul>
<li><p>Скаковых лошадей можно скрещивать только со скаковыми</p></li>
<li><p>Тяжеловесов можно скрещивать с ездовыми только в случае, если они одного окраса</p></li>
<li><p>Ездовых можно скрещивать с пони только если разница в их росте не больше 20см.</p></li>
<li><p>Лошадей внутри одного типа можно скрещивать</p></li>
<li><p>При скрещивании разница в возрасте должна быть не больше 2 лет</p></li>
<li><p>Ну и конечно, их пол должен быть разным.</p></li>
</ul>

<p>Задание: Ваша задача - вывести нужные комбинации.</p>

<p>Формат решения: Вам даны 2 таблицы</p>

<pre><code>create table Horse(
 name text,
 height int, -- рост лошади в сантиметрах
 code varchar(6) PRIMARY KEY, -- код лошади в формате xxx-aa
 color text, -- окрас лошади
 sex text, -- пол лошади в формате &quot;male&quot;/&quot;female&quot;
 age int -- возраст в годах
);

create table Horse_type(
 code varchar(3), -- код лошади в формате xxx
 name text -- тип лошади, один из `Ездовые`, `Тяжеловесы`, `Скаковые`, `Пони`
);

</code></pre>

<p>Вам необходимо написать скрипт, который по этим двум таблицам выведет все возможные комбинации в формате <code>name:text, name:text</code>.</p>
