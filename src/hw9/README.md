          <h1>Эпилог</h1>

<p>По возвращению c практики в приюте тетушки Мейв у главного героя появилась отличная строчка в резюме - теперь он мог найти настоящую работу, где его будут ценить!</p>

<p><em>И правда, предложения не пришлось долго ждать: самый влиятельный в нашем городке человек - Маттиас Феллини, владелец местного футбольного клуба (который кстати перестал быть молодежным), решил связаться со мной. Не помню, что точно он мне сказал, но пояснил, что открывает в городе сеть отелей, и ему очень понравилась мое решение тестового задания. Поэтому он готов предложить мне должность Главного Архитектора Корпорации Феллини!</em></p>

<p>Напоминание - в 1 задании был такой пункт:</p>

<ol>
<li><p><em>Система бронирования отелей База должна содержать следующую информацию:</em></p>

<ul>
<li>Список отелей – название отеля, количество звезд и город, в котором он находится;</li>
<li>Информацию о номерах в отеле – количество человек в номере;</li>
<li>Фамилию, имя и email забронировавшего номер пользователя с указанием периода бронирования.</li>
</ul></li>
</ol>

<p><em>Вообще говоря, требования и задачи на должности оказались заметно выше, чем я ожидал:</em></p>

<h2>Заполнить таблицу (1 балл)</h2>

<p>Заполните (можно вручную) базу следующей информацией:</p>

<ol>
<li>Список отелей - в таблице должно быть не менее 20 разных отелей (как минимум по 4 каждой звезды)</li>
<li>В каждом отеле должны быть как минимум 1-местные и 2-местные.</li>
<li>Должно быть суммарно не меньше 20 человек с различной информацией.</li>
</ol>

<p>Также необходимо реализовать все ключи, индексы для упрощенного поиска, и разные ограничения и правила.</p>

<h2>Написать основные функции (до 2 баллов)</h2>

<p>Далее напишем несколько функций для комфортной работы с базой.</p>

<h3>1. Создать VIEW (1 балл)</h3>

<p>Необходимо создать VIEW, в которой будет сводная информация: Имя человека, дата брони, название отеля, номер (число), количество мест в номере, дата заезда, а также еще какая-нибудь полезная информация (на ваше усмотрение).</p>

<h3>2. Функция вывода свободных номеров (1 балл)</h3>

<p>Необходимо реализовать функцию, которая принимает на вход</p>

<ul>
<li><code>hotel_name::text</code></li>
<li><code>dt_from::timestamp</code></li>
<li><code>dt_to::timestamp</code></li>
</ul>

<p>И возвращает информацию о всех свободных номерах в отеле <code>hotel_name</code> в период <code>[dt_from, dt_to]</code>.</p>

<h2>Написать функции для продвинутой работы (до 3 баллов)</h2>

<h3>Написать функцию оптимального распределения по номерам (3 балла)</h3>

<p>Необходимо реализовать функцию <code>make_distribution</code>, которая принимает на вход</p>

<ul>
<li><code>dates::timestamp[][]</code> -- в виде <code>[[dt_from, dt_to],...]</code> длины <code>n</code> -- даты заезда и выезда.</li>
<li><code>agree::boolean[]</code> -- в виде <code>[bool, bool, ...]</code> длины <code>n</code> -- индикаторы согласия на 2-местные номера.</li>
<li><code>hotel_id::int</code> -- id отеля, в который постояльцы хотят заселиться.</li>
</ul>

<p>И возвращает таблицу вида:</p>

<ul>
<li><code>id::int</code> -- id постояльца, генерируется из списка <code>dates</code> и <code>agree</code>.</li>
<li><code>dt_from::timestamp</code> -- дата заезда.</li>
<li><code>dt_to::timestamp</code> -- дата выезда.</li>
<li><code>room::int</code> -- номер (или id) комнаты в отеле <code>hotel_id</code>.</li>
</ul>

<p>Иными словами, эта функция составляет распределение людей по номерам внутри отеля так, чтобы все были удовлетворены. Если такое не возможно, функция должна выдать пустую таблицу.</p>

<h3>Написать функцию оптимального распределения Ч.2 (4 балла)</h3>

<p>Напишите функцию, которая по таблице</p>

<pre><code> create table guests(
  id int,
  rich_index float, -- индекс &quot;богатости&quot;
  noise_level int -- создаваемый уровень шума
 )
</code></pre>

<p>найдет такое подмноджество гостей, которые суммарно не превысят порог шума в <code>1000</code> единиц, и в то же время принесут наибольшую прибыль (будем считать, что человек платит в день столько денег, сколько у него <code>rich_index</code>). Учтите, что:</p>

<ul>
<li>Каждый из людей приезжает в отель на все лето (с 1 июня по 31 августа, на 92 дня).</li>
<li>Единовременно в отеле может находиться сколько угодно людей, если суммарный уровень шума не превышает порога.</li>
<li>Задание будет засчитано, если найденная максимальная прибыль будет отличаться от оптимальной (которую вы, конечно, не знаете) не более чем на <code>20%</code>.</li>
<li>В качестве ответа функция должна возвращать 3 значения:

<ul>
<li>Максимальная прибыль (приближённое значение).</li>
<li>Массив <code>id</code> гостей, которые максимизируют прибыль.</li>
<li>Соответствующий максимальному варианту уровень шума.</li>
</ul></li>
</ul>

<p><strong>Каждая из задач должна работать не дольше чем 2 минуты для входных данных размером не больше 1000</strong>.</p>
